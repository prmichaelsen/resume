#### AQL VS Code Extension

I identified an opportunity to improve developer experience and productivity with Amazon's internal query language (which is proprietary, so I will refer to it here as AQL) by building a suite of VS Code extensions. Previously, developers lacked native IDE support for AQL, making it challenging to write, troubleshoot, and execute AQL code efficiently within Amazon's environment.

As such, I took on the development of an AQL VS Code Extension Suite in my free time to provide rich support for coding, testing, and executing sandbox queries directly within VS Code.

The query executor enables selecting any AQL query and executing it as a sandbox query directly from within VS Code, then surfacing the results in a query results panel to display a history of executed queries and their results. The extension supports built-in configuration options for sandbox settings.

I additionally built a separate language server extension which supports:

- Syntax highlighting
- Hover view to preview implementation
- Right click a view to "Go to implementations" to open source file in IDE
- Right click a view to "Find all references" to see all usages in code
- Report arity problems with comparison, arithmetic and other built-in operators
- Report basic syntax errors, such as invalid characters or brace mismatch
- Preview relevant views by hovering namespaces or incomplete views
- Completion suggestions for keywords and built in operators
- Completion suggestion for possible views
- Test explorer, execution, reporting, coverage tree view, and inline coverage

I worked with the AQL SDM to help them understand how they can leverage some of my work or use my learnings to inform their OP1 roadmap for 2025 which includes goals for better developer support.

I additionally worked to drive adoption and testing of the tool with other developers, including showcasing the tool during an org-wide HIT demo and presenting the tool at the Amazon Dev Con in Seattle 2025.

##### Impact

The current iteration of the tool improves readability, error detection, and enables faster comprehension of code functionality with its preview and navigation features. The completion suggestions and hover texts help developers learn AQL as they work, instead of by referencing libraries or wikis.

The built-in query executor allows developers to maintain local copies of sandbox queries organized in directories and subdirectories, or even write adhoc queries inline in files they are currently developing in, a major improvement over writing code in a web interface and tracking several browser tabs of queries. Lastly, the test explorer, execution, inline and tree view coverage reporting enable real-time, frictionless feedback while writing tests.

Currently the tool has been adopted by **1000+ weekly users**, with positive feedback such as "inline internal view references and syntax highlighting is an absolute game-changer", "really useful to directly go the implementations", and "It's the best extension I've seen for AQL, and I've been using it every day. It has saved me a lot of time." Per survey data, the tool saves developers on average **3 hours per week**. With 1000+ weekly users saving 3 hours per week, this translates to **156,000 hours saved annually**. Assuming a minimum engineer salary of $150,000 (approximately $72/hour), the tool saves Amazon an estimated **$11.2M per year** in developer productivity.
